Frequently Asked Questions

Does compressing reduce image quality?

By definition, compression for formats like JPG/JPEG is lossy, which discards high-frequency pixel nuances that are generally indiscernible to human eyes. At our default 80% slider level, you'll observe file size decreases up to 75% with virtually zero perceptible changes in visual quality. Transparent PNGs are converted to high-fidelity compressed JPGs automatically.

What is the maximum file size I can compress?

Our client-side system allows processing of files up to 20MB. Because the compression procedures occur in real-time inside your browser's dedicated Javascript thread heap, very large files require a few extra milliseconds to buffer, but there are no constraints on the size or quantities of files loaded!

Is my image uploaded to any server during processing?

Absolutely never! Security remains a fundamental pillar of our development approach. Your files are converted to internal URL object streams and kept static in local ram blocks. When the canvas outputs the byte arrays, it executes the payload as an instant local file download anchor. No database trackers are implemented.

What image formats does the compressor support?

The image compression module completely supports input formats such as JPG, JPEG, standard PNG, and WebP. For formats like PNG that support transparency layers, we preserve translucent parameters unless JPEG targets are specified which substitute trans elements with crisp canvas neutral fills.
